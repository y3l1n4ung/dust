# Roadmap: Dust i18n

Dust i18n is an extraction-first translation system that scans Dart source code
for translation keys and manages Flutter ARB files and generated Dart resolvers.

## Goal

Provide a zero-overhead, type-safe, and high-performance internationalization
workflow that handles extraction, validation, and chunked generation
automatically.

## Public API

Developers use a simple global helper:

```dart
Text(translate(context, 'app.title'))

Text(
  translate(
    context,
    'hello_user',
    args: {'name': user.name},
  ),
)
```

For performance-critical loops, an optimized translator is available:

```dart
final t = translatorOf(context);
Text(t.translate('app.title'))
```

## Features

- [ ] **Tree-sitter Scanner**: Extract keys from `translate()` calls.
- [ ] **ARB Reconciliation**: Merge extracted keys into existing `.arb` files.
- [ ] **Validation**: Detect duplicate keys, dynamic keys, and placeholder mismatches.
- [ ] **Chunked Generation**: Generate feature-based resolver maps instead of one giant switch.
- [ ] **CI Support**: `dust i18n check` to verify sync status.
- [ ] **Migration**: `dust i18n migrate` from JSON or other formats.

## Implementation Plan

1. **Scanner**: Implement a global workspace scanner using Tree-sitter queries.
2. **Reconciler**: Logic to read/write ARB files while preserving existing values.
3. **Plugin**: A new `dust_i18n_plugin` that emits chunked Dart resolvers.
4. **Registry**: A master registry that merges all feature chunks.

## Tests

- Rust tests for the Tree-sitter query extraction.
- Rust tests for ARB merging logic (handling comments, metadata, etc.).
- Dart integration tests in `examples/product_showcase` verifying the
  `translate` runtime behavior.
- Stress test coverage for large numbers of translation keys.
